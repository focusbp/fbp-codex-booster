# Event Registration Sample

Use this sample when a user wants a small event registration demo with both an admin side and a public side.

## Scope

The sample is intentionally independent from `project`, support tickets, external calendars, email delivery, LINE, Square, or any production app.

It includes:

- Admin event session management.
- Admin public registration URL button and dialog.
- Admin participants side panel per event session.
- Admin participant add/delete from the side panel.
- Admin registration status change.
- Public list of open sessions.
- Public registration form.
- Public completion page.
- Dynamic seed event sessions.

It excludes:

- Project linkage.
- Public registration history.
- Public cancellation.
- External calendar sync.
- Email sending.
- Payment or authentication.

## Fixed Names

- Session table: `event_sessions`
- Registration table: `event_registrations`
- Admin class: `event_sessions_original_management`
- Public class: `event_registration_public`
- Installer: `scripts/install_event_registration.php`
- Manifest: `assets/event-registration/event-registration.json`
- Session status constant array: `event_session_status`
- Registration status constant array: `event_registration_status`

The public class is separate from `public_pages` so the sample can be installed into projects that already have a `public_pages` class. In a production app, moving the entry functions into `public_pages` is acceptable.

## Admin Flow

`event_sessions_original_management::run` shows the session list.

The toolbar includes:

- `Public Registration URL`: opens a dialog with the public URL generated by `$ctl->get_APP_URL("event_registration_public", "page")`.
- `Add Session`: opens the session add dialog.

Each session row includes:

- A `Participants` button that opens a side panel with public-side registrations.
- In the participants side panel, admins can add a participant manually, update status, or delete a participant.
- Edit.
- Delete, only when the session has no active registrations.
- Registrations, which opens a dialog showing submitted registrations for that session.

Registration status changes are admin-only. The public side does not show existing registrations and does not allow cancellation.

## Public Flow

`event_registration_public::page` shows only event sessions:

- `status = open`
- not in the past
- active registrations are below capacity

Visitors select a session, open `event_registration_public::form`, enter name/email/phone/message, and submit to `event_registration_public::save`.

The public session parameter is encrypted with `$ctl->encrypt()` in the generated URL and decrypted by the public class.

## Installer

From an FBP project root:

```bash
php fbp/docs/.agents/skills/fbp-app-samples/scripts/install_event_registration.php
```

Options:

- `--root=/path/to/project`: run outside the project root.
- `--keep-data`: keep existing `event_sessions` and `event_registrations` data.
- `--no-copy`: update DB/constant data without copying code.

The default installer copies code, upserts constants/tables/fields, resets registration demo data, and inserts future seed sessions plus one seed registration.

## Verification

After installation, run:

```bash
php fbp/cli.php app_call --json='{"class":"event_sessions_original_management","function":"run"}'
php fbp/cli.php app_check --json='{"class":"event_sessions_original_management","function":"run","checks":[{"path":"response_json.work_area.html","contains":"Public Registration URL","label":"public url button"},{"path":"response_json.work_area.html","contains":"Event Sessions:","label":"session list"}]}'
php fbp/cli.php app_call --json='{"class":"event_sessions_original_management","function":"public_url_dialog"}'
php fbp/cli.php app_call --json='{"class":"event_registration_public","function":"page"}'
php fbp/cli.php data_list --json='{"table":"event_sessions","max":20}'
php fbp/cli.php data_list --json='{"table":"event_registrations","max":20}'
```

To test a public registration end to end, use the browser or fetch one available `event_sessions` ID, open the generated public page, select a session, and submit the form. For CLI-only verification, add a row with `data_add` and confirm it appears in `data_list`.
